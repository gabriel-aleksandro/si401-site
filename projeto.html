Resumo

O Jogo da Memória a ser implementado consiste em um tabuleiro M×N (quadrado entre 2×2 e 8×8, número par de peças). Versão single-player inspirada no jogo clássico, com duas modalidades: Clássica e Contra o Tempo. A plataforma será online e utilizará as tecnologias estudadas na disciplina (HTML, CSS, JavaScript e PHP). O sistema deverá manter histórico de partidas e permitir autenticação de usuários.

Regras do Jogo

Em cada jogada o jogador revela duas peças. Se forem iguais, as peças permanecem reveladas e o jogador acumula ponto; caso contrário, as peças retornam a ficar ocultas.

No modo multiplayers ganha quem tiver mais pontos quando todas as peças são reveladas. Na variante single player, a pontuação final é o número de jogadas necessárias para revelar todo o tabuleiro (quanto menor, melhor).

Especificações do Projeto

Configurações

  • O usuário poderá escolher dimensões do tabuleiro (quadrado): mínimo 2×2, máximo 8×8, sempre número par de peças.
  • Modalidades disponíveis: Clássica e Contra o Tempo.

Modalidade Clássica

Sem cronômetro: conta-se o número de jogadas até o tabuleiro ser totalmente revelado. Objetivo: obter o menor número de jogadas.

Modalidade Contra o Tempo

Similar à clássica, porém com um timer. O jogador deve terminar antes do tempo se esgotar; caso contrário, a partida é contabilizada como derrota. O tempo disponível por partida deve depender do tamanho do tabuleiro.

Histórico de Partidas

O sistema deve manter um histórico com, para cada partida:

  • Nome do jogador;
  • Dimensões do tabuleiro;
  • Modalidade da partida;
  • Tempo gasto até vitória/derrota (contado a partir do primeiro clique);
  • Número de jogadas;
  • Resultado (vitória ou derrota);
  • Data e hora da partida.

Botões de "Trapaça"

Devem existir dois botões que alterem apenas a exibição do tabuleiro:

  1. Mostrar todas as peças ocultas (sem afetar a lógica do jogo).
  2. Restaurar exibição normal (manter reveladas as peças já descobertas; ocultar as demais).

Desistência

O jogador pode desistir a qualquer momento e retornar à tela inicial de configurações.

Implementação do jogo

O jogo deve ser implementado integralmente em JavaScript (front-end). A plataforma deve ainda contar com um back-end em PHP para autenticação, persistência do histórico e geração do ranking.

Front-end (requisitos)

  • A primeira tela exibida: formulário de login (usuário + senha) e link para a página de cadastro.
  • Página de cadastro: solicitar nome completo, data de nascimento, CPF, telefone, e-mail, username (único) e senha.
  • Após autenticação, o usuário é redirecionado para a página do Jogo da Memória (implementado em JavaScript).
  • A página do jogo deve mostrar, no mínimo:
    • Tempo da partida (se iniciada);
    • Número de jogadas;
    • Configuração do tabuleiro;
    • Modalidade selecionada;
    • Tempo restante (no modo Contra o Tempo);
    • Histórico do usuário (conforme campos descritos anteriormente);
    • Botões de ativação/desativação do modo Trapaça;
    • Links para: ranking global, edição de informações pessoais e opção para desconectar.
  • Na página de edição de usuário: data de nascimento, CPF e username não podem ser alterados.

Back-end (requisitos)

Deve ser implementado em PHP, com dados armazenados em MySQL ou MariaDB. Responsabilidades:

  • Gerenciar autenticação de jogadores (uso de sessões PHP);
  • Armazenar e recuperar resultados de partidas;
  • Gerar ranking dos 10 melhores jogadores conforme regras abaixo.

Ranking global (regras)

Exibir usernames e dados de partida para 10 jogadores, escolhendo as partidas disputadas (e vencidas) nos 10 maiores tabuleiros. Se houver empate por dimensão, exibir a partida com menor número de jogadas.

Estrutura de dados

As tabelas do SGBD (usuários, partidas, etc.) ficam a critério do grupo, mas é obrigatório entregar um script PHP separado (ou instruções claras) para criar as tabelas, acompanhado de arquivo-texto com instruções de uso.

ATENÇÃO: o uso de frameworks PHP não é permitido.

Parciais e Entregas

Parcial 1

Versões não-funcionais das páginas: todas as páginas do sistema presentes, formatadas com CSS externo e hyperlinks funcionando. Componentes dinâmicos substituídos por imagens ou valores estáticos (ex.: imagem do tabuleiro, tabela de ranking com valores fixos).

Parcial 2

Toda a dinâmica do Jogo da Memória implementada em JavaScript (front-end), mas sem funcionalidades de back-end: jogo jogável localmente, sem persistência nem controle de acesso.

Parcial 3

Versão final com integração completa front-end e back-end (autenticação, persistência, ranking, histórico).

Entrega de cada parcial

Cada entrega deve conter:

  • Arquivo ZIP com todos os arquivos necessários (.html, .js, .php, .css, imagens etc.) mantendo a estrutura de diretórios;
  • Arquivo TXT/PDF com instruções de configuração e link para vídeo de apresentação;
  • Arquivo .XLSX com detalhamento da contribuição de cada membro (modelo disponível no curso).

Os vídeos de apresentação (máx. 10 minutos) devem ser hospedados em serviço gratuito (ex.: YouTube) e link fornecido junto à entrega.

Regras para o Trabalho

  • Grupos de 4 a 5 alunos.
  • O projeto corresponde a 75% da média do aluno (conforme Plano de Ensino).
  • Todo o código JavaScript deve estar em arquivos .js externos.
  • Folhas de estilo devem ser preferencialmente externas (CSS).
  • Documentos HTML e CSS devem seguir o padrão mais recente e passar na validação do W3C.
  • Cópias de outros grupos ou da internet serão penalizadas.
  • Trabalhos entregues fora do prazo não serão aceitos.

Observações importantes

ATENÇÃO 1: Todas as páginas do front-end devem usar HTML5 e CSS (preferencialmente arquivos CSS externos) e serem validadas em:

  • Validador HTML: https://validator.w3.org/
  • Validador CSS: https://jigsaw.w3.org/css-validator/

ATENÇÃO 2: Não é recomendado o uso de templates CSS; caso optem por usá-los, informem explicitamente no site e forneçam links das fontes.

ATENÇÃO 3: Para o desenvolvimento do jogo, estude eventos temporizados em JavaScript (timers).

Material adicional / Entrega

Ao empacotar a entrega, lembre-se de incluir um README (ou arquivo de instruções) explicando:

  • Como configurar o ambiente (por exemplo: servidor local PHP + MySQL/MariaDB);
  • Como restaurar o banco de dados (script SQL/PHP para criação de tabelas);
  • Links para vídeos de apresentação;
  • Arquivo .XLSX com contribuição dos membros.
